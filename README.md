# Schizophrenia Manual Annotation Dataset

This repository contains the supplementary materials accompanying our study on linguistic patterns in schizophrenia. It provides the manually annotated corpus, annotation guidelines, preprocessing resources, analysis scripts, and experimental results used throughout the study. The repository is intended to facilitate transparency, reproducibility, and future research on computational linguistics, natural language processing, and psychiatric language analysis.

## Repository Overview

The repository includes:

- **Manually annotated datasets** used for the linguistic analyses.
- **Annotation guidelines** describing the complete annotation protocol.
- **Filtering resources** used during corpus construction.
- **Analysis notebooks** for preprocessing, feature extraction, statistical analysis, and feature selection.
- **Experimental results** generated from statistical analyses and machine learning experiments.

## Contents

### Data

The `Data` directory contains the manually annotated datasets used during the study.

- **stage1.csv** — annotations from the first annotation stage.
- **stage2.csv** — annotations from the second annotation stage.

These files contain the linguistic annotations described in the accompanying annotation guide.

### Annotation Guide

The `Tagging_Guide` directory contains the complete annotation manual used during corpus annotation.

The guideline documents:

- annotation principles;
- linguistic categories;
- syntactic and semantic labels;
- annotation conventions;
- examples of annotated texts.

Researchers wishing to extend or reproduce the annotation procedure should consult this document first.

### Regex Filters

The `Regex_Filters` directory contains the regular expression resources employed during corpus construction to identify or exclude posts according to:

- age declarations;
- gender declarations;
- nationality declarations;
- psychiatric disorders.

These filters were used as part of the preprocessing pipeline before manual annotation.

### Code

The notebooks in the `Code` directory reproduce the analyses presented in the paper.

| Notebook | Purpose |
|----------|---------|
| `regex_search.ipynb` | Applies the filtering rules used during corpus construction. |
| `word_counter.ipynb` | Computes descriptive corpus statistics. |
| `inferential_analysis.ipynb` | Performs the inferential statistical analyses. |
| `boruta_run.ipynb` | Executes Boruta feature selection and Random Forest analyses. |

## Results

The `Results` directory contains the outputs generated by the analysis scripts, including:

- Boruta feature selection results;
- Random Forest performance summaries;
- normality tests;
- Mann–Whitney U tests;
- statistically significant variables identified during the analyses.

These files correspond to the outputs reported in the associated publication.

## Requirements

Python dependencies can be installed using

```bash
pip install -r requirements.txt
```

Most analyses were developed using Jupyter Notebooks.

## Reproducibility

The repository has been organized to facilitate full reproducibility of the reported analyses. Running the notebooks in the `Code` directory using the provided datasets should reproduce the statistical analyses and feature selection procedures described in the manuscript.
